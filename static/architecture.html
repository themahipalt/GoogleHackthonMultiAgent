Layer 4 — Orchestrator Agent (Agentic Loop)

Orchestrator — Gemini 2.5 Flash

Agentic loop: sends user message → executes tool calls → feeds results back → repeats until Gemini produces a text-only response (no more tool calls)

  • Max 8 hops (prevents infinite loops)
  • Parallel tool calls in a single hop
  • Yields SSE events at each step
  • Persists tool call logs to Firestore
gemini-2.5-flash agents/orchestrator.py

System Prompt

Instructs Gemini to:

  • Always call tools (never fabricate)
  • Batch independent calls
  • Handle multi-step workflows
  • Return a concise summary

Includes today's date for relative date resolution

SSE Payload Format

Every yield is a JSON event:

{
  "agent": "calendar_agent",
  "msg":   "→ calendar_create_event(...)",
  "ts":    1743436800.0
}

Request Flow — "Schedule standup tomorrow 9am + create prep task"

1
Browser sends GET /stream?message=...&user_id=demo via EventSource. FastAPI returns a StreamingResponse (text/event-stream).
2
Orchestrator yields SSE: {"agent":"orchestrator","msg":"Processing: ..."} immediately so the UI shows activity.
3
Gemini 2.5 Flash receives the message + all 11 tool schemas. It responds with two parallel function_call parts: calendar_create_event and task_create.
4
Orchestrator loop (Hop 1) iterates the function_calls, yields a trace line per call, dispatches each to the tool registry, and logs to Firestore's agent_logs collection.
5
Calendar Agent resolves "tomorrow" → 2026-04-01, calls Firestore (or Google Calendar API if configured), gets back {"created":true,"event_id":"..."}.
6
Task Agent creates the prep task in Firestore (or Google Tasks API), returns {"created":true,"task_id":"..."}.
7
Both results are bundled into a single chat.send_message(response_parts) call. Gemini sees both outcomes and produces a final text summary (no more function_calls → loop breaks).
8
Orchestrator yields the summary text and ✓ Done.. Browser closes the EventSource, renders the response, and refreshes the sidebar via GET /tasks and GET /events.
